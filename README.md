# Bill4Print 项目说明文档

## 1. 项目概述

Bill4Print 是一个用于打印商品标签的自动化工具，主要用于处理电商订单中的商品信息，生成并打印标准化的商品标签。该工具特别适用于农产品、干货等需要分装销售的场景，能够根据订单信息自动生成包含成分、规格、净重、保质期、产地、等级、储存方式等信息的标签。

### 主要功能：
- 自动读取 Excel 订单文件
- 根据商品配置信息生成标签数据
- 支持组合商品（子项）的处理
- 按成分、规格、净重、等级排序标签
- 自动打印标签
- 备份处理过的订单文件
- 支持自定义打印日期

## 3. 系统要求

- Windows 操作系统
- Python 3.x
- 所需 Python 库：
  - pandas
  - openpyxl（用于读取 Excel 文件）
  - psutil（用于进程管理）
- 打印机（支持标签打印）

## 4. 安装说明

1. **安装 Python**：
   - 从 [Python 官网](https://www.python.org/) 下载并安装 Python 3.x
   - 确保勾选 "Add Python to PATH"

2. **安装依赖库**：
   ```bash
   pip install pandas openpyxl psutil
   ```

3. **配置项目**：
   - 修改 `config.json` 文件中的配置项（详见配置说明部分）
   - 编辑 `goods.xlsx` 文件，添加商品信息（详见商品配置部分）

4. **准备打印机**：
   - 确保打印机已正确连接并安装驱动
   - 调整打印机设置，确保标签尺寸正确（默认标签尺寸：50mm × 40mm）

## 6. 使用方法

### 6.1 基本使用

1. **准备订单文件**：
   - 在 `orderExcelDirectory` 配置的目录中放置 Excel 订单文件
   - 订单文件应包含 "规格ID" 和 "SKU件数"（或 "数量"）字段

2. **运行程序**：
   ```bash
   python main.py
   ```

3. **程序执行流程**：
   - 检查并创建锁文件，防止多实例运行
   - 读取配置文件和商品配置
   - 遍历订单目录中的 Excel 文件
   - 处理每个 Excel 文件，生成标签数据
   - 排序标签数据
   - 打印标签
   - 备份并删除处理过的订单文件
   - 每 10 秒检查一次是否有新的订单文件

### 6.2 自定义打印日期

如需指定打印日期（而非使用当前日期），可使用 `--diy_date` 参数运行程序：

```bash
python main.py --diy_date
```

运行后，程序会提示输入自定义日期，格式为 YYYY-MM-DD：

```
请输入自定义日期，格式为YYYY-MM-DD: 2026-02-01
您输入的日期是: 2026-02-01
```

## 7. 核心功能详解

### 7.1 订单处理流程

1. **读取订单文件**：使用 pandas 读取 Excel 订单文件
2. **匹配商品信息**：根据订单中的规格ID匹配 goods.xlsx 中的商品信息
3. **处理组合商品**：如果商品有子项，根据子项配置生成相应数量的标签
4. **排序标签**：按成分、规格、净重、等级排序标签
5. **生成标签数据**：构建包含商品详细信息的标签数据
6. **打印标签**：调用 ddPrint 模块打印标签
7. **备份订单**：将处理过的订单文件备份到 bak 目录

### 7.2 标签格式

标签尺寸：50mm × 40mm

标签内容布局：

| 位置 | 内容 | 示例 |
|------|------|------|
| 顶部 | 数量（仅当数量 > 1 时显示） | 2 |
| 第一行 | 成分 | 成分：新疆若羌红枣 |
| 第二行 | 规格 | 规格：散装称重 |
| 第二行右侧 | 净重 | 净重：250克 |
| 第三行 | 产地 | 产地：新疆 |
| 第三行右侧 | 保质期 | 保质期：12个月 |
| 第四行 | 包装日期 | 包装日期：2026-02-01 |
| 第四行右侧 | 等级 | 等级：特级 |
| 第五行 | 储存方式 | 储存方式：密封冰箱冷藏 |
| 第六行 | 销售商信息 | 销售商：福建省泉州市保康滋补商行 |
| 第七行 | 联系电话 | 电话：18396188860（微信同号） |
| 第八行 | 温馨提示 | 温馨提示：本店所受农产品属于初级农产品，代客分装，散装称重，包装是为了方便运输赠送。 |

### 7.3 组合商品处理

对于组合商品（如 "红枣250克+桂园干250克"），系统会：
1. 识别组合商品的子项配置（格式：子项规格ID#数量,子项规格ID#数量）
2. 根据子项配置和订单数量生成相应数量的子项标签
3. 例如：如果订单数量为 2，子项配置为 "A#1,B#1"，则会生成 2 个 A 标签和 2 个 B 标签

## 8. 故障排除

### 8.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 程序提示 "Another instance of the script is already running" | 程序已经在运行 | 检查是否有其他实例正在运行，或删除 application.lock 文件后重新启动 |
| 程序提示 "Error: File config.json not found" | 配置文件不存在 | 确保 config.json 文件存在且格式正确 |
| 程序提示 "SKU 名称没有在 GoodsConfig 中找到" | 订单中的规格ID在 goods.xlsx 中不存在 | 检查订单中的规格ID是否正确，或在 goods.xlsx 中添加相应的商品信息 |
| 标签打印不正确 | 打印机设置不正确 | 检查打印机设置，确保标签尺寸正确（50mm × 40mm） |

### 8.2 日志和调试

- 程序运行时会在控制台输出处理过程和错误信息
- 处理过的订单文件会备份到 bak 目录，可用于排查问题
- 每次打印的标签数据会保存到 `latest_print_file.xlsx` 文件中，可用于查看打印内容

## 9. 代码结构

### 9.1 主要函数

| 函数名 | 描述 | 参数 | 返回值 |
|-------|------|------|--------|
| `read_config()` | 读取配置文件 | file_path: 配置文件路径 | 配置字典或 None |
| `DoXHS()` | 处理订单文件 | AExcelFile: 订单文件路径 | 布尔值，表示处理是否成功 |
| `main()` | 主函数，协调各模块工作 | 无 | 布尔值，表示程序是否正常运行 |
| `check_lock_file()` | 检查锁文件，防止多实例运行 | lock_file_path: 锁文件路径 | 布尔值，表示是否可以创建新实例 |
| `create_lock_file()` | 创建锁文件 | lock_file_path: 锁文件路径 | 无 |
| `remove_lock_file()` | 删除锁文件 | lock_file_path: 锁文件路径 | 无 |

### 9.2 处理流程

1. **初始化**：检查锁文件，读取配置文件和商品配置
2. **订单处理**：遍历订单目录，处理每个订单文件
3. **标签生成**：根据订单信息和商品配置生成标签数据
4. **标签排序**：按成分、规格、净重、等级排序标签
5. **标签打印**：调用打印模块打印标签
6. **文件备份**：备份并删除处理过的订单文件
7. **循环检查**：每 10 秒检查一次是否有新的订单文件

## 10. 注意事项

1. **订单文件格式**：订单文件必须是 Excel 格式（.xlsx 或 .xls），并包含 "规格ID" 和 "SKU件数"（或 "数量"）字段

2. **商品配置**：
   - 确保 goods.xlsx 文件中的 "规格ID" 与订单文件中的一致
   - 组合商品的子项配置格式必须正确（子项规格ID#数量,子项规格ID#数量）

3. **打印机设置**：
   - 确保打印机已正确连接并安装驱动
   - 调整打印机设置，确保标签尺寸正确（50mm × 40mm）

4. **备份文件**：处理过的订单文件会被备份到 bak 目录，建议定期清理该目录，避免占用过多磁盘空间

5. **多实例运行**：程序会创建锁文件防止多实例运行，如需同时运行多个实例，需修改锁文件路径
